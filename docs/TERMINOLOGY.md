# Словарь публичных терминов

Единая система названий для интерфейса, документов, видео и презентации. Технический raw-термин
допустим только как вторичная подпись (моноширинный код, tooltip) там, где он нужен для
доказательства.

| Публичный термин | Что означает | Raw-термин (где допустим) |
|---|---|---|
| ГигаАгент | Агентный контур, выполняющий генеративную работу; при первом техническом упоминании — «ГигаАгент (Ouroboros)» | `Ouroboros v6.61.4` — в архитектуре и диагностике |
| Модель | Языковая модель маршрута агента | `openrouter::z-ai/glm-5.2` — в диагностике/учёте |
| Живая генерация / живой запуск | Новый реальный прогон агента, а не показ сохранённого | `live_ouroboros` — вторичный код режима |
| Сохранённый результат | Показ ранее захваченного результата живого запуска | `replay` — вторичный код режима |
| Резервный шаблон | Детерминированный не-LLM артефакт при недоступности агента | `deterministic_template` — вторичный код режима |
| Проверка качества / 22 автоматические проверки | Детерминированный реестр проверок фактов, ссылок, политик | `QA`, идентификаторы проверок — в коде и отчётах |
| Привязка «утверждение → факт → источник» | Двусторонняя проверка каждого числа/даты/ссылки | `claim_evidence` — в данных |
| Замечание | Комментарий редактора к конкретному полю | `feedback` — в API |
| Точечная версия / доработка | Новая версия только по разрешённым полям, с diff | `revision`, `CommunicationPatch` |
| Правило | Ограниченное декларативное правило с тестами и человеческим утверждением | `rule_proposal`, `rule_version_id` |
| Тестовое действие | Утверждение, выполненное в тестовом режиме, не submission-решение | `test_only` — вторичная метка |
| Идентификатор запуска / задачи | Публичный идентификатор для трассы и доказательств | само значение ID |
| Контрольная сумма (манифеста) | SHA-256 артефакта/манифеста | raw SHA — в отчётах |
| Агентный контур | Среда исполнения агента (закреплённая версия) | `runtime` — в техдокументации |
| Письмо | Канал e-mail | поле API `email` |
| СМС | Канал SMS | поле API `sms`; `UCS-2`/сегменты — в метриках |
| Зафиксированные доказательства | Неизменяемый checksummed-пакет результатов | `frozen evidence` — только в инженерных документах |
| Только синтетические данные; внешняя отправка отключена | Границы безопасности продукта | литеральный токен `synthetic-only/no-send` сохраняется один раз в технической части README для contract-теста |

Не переводить общеупотребительные URL, UTM, API, MCP и ZIP — объяснить при первом упоминании и
использовать последовательно. MCP — протокол инструментов модели; ровно два инструмента задачи:
«получить контекст» (`mcp_factory__cf_context_get`) и «сохранить черновик»
(`mcp_factory__cf_draft_save`).
