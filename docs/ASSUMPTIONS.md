# Допущения и ограничения

## Что фактически есть в прототипе

- Продукт строго synthetic-only/no-send: все данные синтетические, внешняя отправка отсутствует
  как функция, финальные решения принимает человек.
- Два профиля запуска. Локальный Docker Compose — воспроизводимый профиль для проверки по README
  (loopback-шлюз с Basic Auth). Hosted-профиль Railway — общий стенд команды с отдельной страницей
  входа и подписанной сессией в `HttpOnly` cookie; наружу открыт только шлюз.
- Маршрут модели — `openrouter::z-ai/glm-5.2` внутри закреплённого ГигаАгента
  (Ouroboros `v6.61.4`); автоматической смены модели или провайдера нет, backend LLM не вызывает.
- Совместимость строгого контракта инструментов с GLM-5.2 подтверждена десятью сохранёнными
  живыми кейсами Basket-03 (B01, B02, B03, B04, B06, B07, B09, B10, B14, B15) с фактическими
  СМС/письмами, полным учётом токенов и детерминированной оценкой 100/100.
- Живая генерация занимает от двух до пяти минут на кейс (фактические значения — в
  `reports/basket03-mvp-testing/summary.csv`). Прежний внутренний ориентир «до 30 секунд»
  относился к инженерному профилю ранних этапов и не является требованием жюри; hosted-профиль
  использует увеличенные тайм-ауты задач.
- СМС/письма, проверка качества, доработка, управляемый жизненный цикл правил,
  утверждение/экспорт и детерминированное поведение каталога B01–B15 реализованы и покрыты
  тестами.
- Счётчики проекта учитывают только собственный расход; общий остаток аккаунта провайдера
  неизвестен и не заявляется.

## Baseline и гипотезы — не достигнутый эффект

Величины из Proposal — «1–2 часа до первого черновика», «−40–60% времени подготовки»,
«−30–50% ручных операций», «+10–20% отклика» — остаются baseline-допущениями и гипотезами.
Прототип их не измерял и не подтверждал. Будущий пилот обязан заранее определить популяцию,
baseline, рубрику ревью, таксономию ошибок и окно измерения, прежде чем заявлять эффект.

## Сознательные ограничения

- Нет production-данных, интеграций доставки и multi-tenant авторизации; hosted-стенд защищён
  одной командной учётной записью и предназначен для демонстрации.
- Чат, голос, CLM, баннеры и новые навыки агента относятся к выключенному контуру P1/P2 и в
  текущий конкурсный релиз не входят.
- Резервный шаблон и показ сохранённых результатов поддерживают разработку и честную
  демонстрацию; ни один из них не выдаётся за живую генерацию.
- Технический протокол Basket-03 зафиксирован со статусом «неканоническое доказательство»
  (13/15; B05 и B08 неуспешны): это честный отчёт тестирования MVP, а не frozen release
  evidence. Каноническая полная корзина, независимые человеческие оценки и два sign-off
  остаются шагами подготовки финальной сдачи. Озвученное демо-видео опубликовано; основная и
  запасная ссылки приведены в `README.md` и `docs/JURY_GUIDE.md`.
