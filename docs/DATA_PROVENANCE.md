# Происхождение данных

## Что лежит в каталоге и откуда

Приложение использует только фиксированный синтетический каталог `data/synthetic/`:

| Путь | Содержимое | Происхождение |
| --- | --- | --- |
| `products/products.json` | Шесть вымышленных продуктов: факт-карточки и леджеры | Новые фикстуры по каноническому контракту |
| `segments/personas.json` | Девять агрегированных неперсональных персон | Новые fixed-seed фикстуры |
| `touch_history/history.json` | Вымышленные агрегированные истории касаний | Новые синтетические фикстуры |
| `policies/policies.json` | Versioned-политики контактов/права/каналов/концептов | Новые детерминированные фикстуры |
| `cases/gate1.json` | Точная корзина B01–B15 | Проекция ожидаемых кейсов из ТЗ |
| `evaluation/*.json` | Assertions бизнес/chaos-кейсов и выбор ревью | Проекция release-целей из ТЗ |

Каждая запись несёт `synthetic=true`; продукты, бренды, компании и условия вымышленные, CTA-хосты
используют только зарезервированные домены `.test`/`.invalid`. Каталог намеренно не содержит
имён, телефонов, e-mail, налоговых и счётных идентификаторов, а также чувствительных или
дискриминационных признаков.

## Как проверить отсутствие реальных ПДн

- `make security` валидирует флаги/URL каталога и сканирует сгенерированные артефакты на
  PII/внутренние данные, а также дерево и историю Git на секреты;
- `make source-audit` проверяет, что приватные материалы не отслеживаются Git и не попадают в
  Docker-контексты;
- исходные материалы хакатона лежат read-only в ignored `private_sources/` и никогда не входят в
  код, промпты, фикстуры, evidence или submission; legacy-код не копировался — переиспользованные
  идеи переписаны заново (см. [Legacy reuse](LEGACY_REUSE.md)).

## Captured / replay / reconstructed

- **Captured (live)** — результат нового реального прогона Ouroboros: сохранённый пакет с
  task/tool-квитанциями, режимом `live_ouroboros` и checksummed-артефактами. Только такие записи
  считаются live-доказательствами.
- **Replay** — показ ранее захваченного immutable live-артефакта: проверяется манифест/checksum,
  отображается крупный бейдж и source run ID. Replay не создаёт новую задачу и не входит в
  live-счётчики и латентность.
- **Reconstructed / deterministic template** — детерминированно собранный без LLM артефакт
  (разработка или резервный режим). Всегда помечен своим режимом и никогда не выдаётся за результат
  работы Ouroboros.

Захваченные живые результаты хранятся в обезличенном пакете
[reports/basket03-mvp-testing](../reports/basket03-mvp-testing): десять подтверждённых живых
кейсов с фактическими СМС/письмами (включая B09 — только письмо и B10 — только СМС), плюс честно
сохранённые неуспешные B05/B08. Пакет связан контрольными суммами (`checksums.sha256`, SHA-256
исходного отчёта в его README) и не содержит сырых transport-payload, идентификаторов
provider-запросов и секретов. Канонического frozen-баскета 15/15 в `artifacts/` нет: Basket-03 —
отчёт тестирования MVP, а не release evidence. Любые новые данные или ассеты требуют новой строки
провенанса, проверки синтетичности/приватности и подтверждения совместимой лицензии до
использования.
