# Безопасность

## Ключевые границы

- **Секрет только в Ouroboros.** `make init` принимает ключ OpenRouter скрытым вводом либо через
  внешний файл с mode `0600`; ключ остаётся вне Git checkout. Локальный Compose монтирует этот файл
  read-only, supervisor переносит значение в приватный runtime-файл и очищает своё окружение до
  запуска дочерних процессов. Hosted-профиль принимает секрет из Railway с теми же границами.
  Gateway, app, браузер и host-процессы ключ не получают; `.env.local` содержит только путь к
  внешнему файлу. `.dockerignore` исключает ключи, runtime и операторские файлы из build-контекста.
- **Нет отправки.** SMTP/SMS/CRM-адаптеров не существует как кода; «утверждение» означает только
  ревью человеком. Экспорт — локальный ZIP.
- **Backend без LLM.** В образе и конфигурации app нет provider-SDK и ключей; архитектурный скан
  (`make security`, отдельный architecture-тест) падает при их появлении.
- **Только синтетика.** Каталог, персоны, политики и кейсы вымышленные, домены зарезервированные
  (`.test`/`.invalid`); происхождение данных — в [Data provenance](DATA_PROVENANCE.md).
- **Редакция на границе логов.** Safe-события не содержат prompt-текста, chain-of-thought,
  provider-ответов, Authorization-заголовков и секретов; Playwright-трейсы очищаются от
  Basic-Auth заголовков до попадания в evidence.
- **Immutable evidence.** Evidence-каталоги checksummed и после финализации не изменяются;
  сканер проверяет архивы без распаковки (zip-slip, symlink, секреты, PII, внутренние URL).

## Сеть и доступ

Два профиля доступа. В локальном Docker Compose наружу опубликован только `127.0.0.1:8080`
gateway с Basic Auth, CSP, no-sniff, frame- и permissions-заголовками; доступ локальный или через
SSH-туннель. В hosted-профиле Railway тот же продукт закрыт отдельной страницей входа: подписанная
ограниченная по времени сессия в `HttpOnly`/`Secure` cookie, CSRF-проверка изменяющих запросов,
без сессии HTML уходит на `/login`, API отвечает `401`; открыт только `/healthz` для health
check платформы (см. [Railway runbook](RAILWAY.md)). В обоих профилях сети app и runtime
приватные; provider-egress есть только у Ouroboros. Сервисы non-root, с capability-drop,
`no-new-privileges`, лимитами CPU/памяти/PID и ротацией логов.

## Модель угроз и отказов

| Угроза / отказ | Защита | Проверка |
| --- | --- | --- |
| Prompt-injection в брифе, истории, замечаниях | Инструкция отделена от данных; все бизнес-блоки помечены untrusted; текст в данных не расширяет возможности | Кейс B14, security-тесты |
| Расширение возможностей агента | Provider-запрос содержит ровно два factory-инструмента; полный `disabled_tools` действует на discovery и execution | No-forward probe, contract-тесты, lock |
| Выдуманные факты/цифры | Двусторонний grounding: заявленные evidence + независимое извлечение чисел/дат/сумм/URL; blocker закрывает утверждение | QA-registry (22 проверки), B05 |
| Утечка секрета | Ключ вне checkout; mount только в Ouroboros; сканы дерева, истории Git, образов и артефактов | `make security`, image-скан |
| Небезопасный HTML/XSS | Модель не возвращает raw HTML; детерминированный sanitizer/renderer; sandbox превью | Тесты рендерера, e2e |
| Подмена live-доказательств | Режимы `live_ouroboros`/`replay`/`deterministic_template` фиксируются в записях и UI-бейджах; replay не входит в live-метрики | Evidence-манифесты, UI-сценарии |
| Обход человеческого утверждения | Approve/reject/rollback требуют аутентифицированной человеческой сессии; агентский токен не имеет этих прав; автоматические события помечены `test_only=true` | API/E2E-тесты |
| Поздняя запись после терминала | MCP-авторизация одноразовая: поздний save после fallback закрыт, идемпотентный повтор возвращает исходную квитанцию | Chaos X02/X04/X05 |
| Zip-slip / path traversal / SSRF | Серверные имена файлов и checksums в экспорте; внешние URL никогда не запрашиваются задачей | Security-тесты, сканер архивов |

`make security` прогоняет сканы дерева/истории/артефактов/no-send/сети/архитектуры и
lock-bound инвентаризацию зависимостей и лицензий; provider-вызовов команда не делает.
Ограничение: реальные клиентские данные и production-отправка запрещены на уровне продукта.
