# Ключевые решения

1. Все данные и политики синтетические; внешняя отправка отсутствует как функция.
2. СМС и письма создаёт закреплённый Ouroboros, backend LLM не вызывает.
3. Каждая задача видит ровно два MCP-инструмента: получить контекст и сохранить черновик.
4. Backend независимо проверяет факты и политики, хранит неизменяемые версии и блокирует
   утверждение при нарушении.
5. Замечание создаёт видимую новую версию; переиспользуемое правило тестируется и активируется
   только человеком.
6. Публичный репозиторий является очищенным конкурсным срезом; внутренняя история автономной
   разработки хранится отдельно и для запуска продукта не нужна.
